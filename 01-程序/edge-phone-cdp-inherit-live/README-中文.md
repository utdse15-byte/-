# Edge 手机 CDP 控制器 v6.8.6

这是一个面向 Windows Edge 与 Android 手机的本地网页控制器。Edge 继续运行在 Windows 上，手机浏览器只负责显示远程网页画面，并把用户本人实时产生的点击、滚动、输入、标签切换和文件选择操作发送回电脑。

本项目的定位是**真人实时远程操作**，不是无人值守自动化、爬虫、批量提问、批量抓取回答、验证码处理或平台限制绕过。

## 1. 本项目保留的不可变运行条件

### 原 Edge 配置

控制器直接使用 Windows 中原来的 Edge 配置，继承：

- Cookie 和网站登录状态；
- 扩展；
- 收藏夹；
- 网站权限；
- 浏览历史；
- 已有会话。

默认不会创建隔离的 `--user-data-dir`。启动脚本会读取上次使用的 `Default`、`Profile 1` 等真实配置目录。

### 固定本地代理

默认并明确保留以下网络参数：

```text
--proxy-server="http://127.0.0.1:7897"
--host-resolver-rules="MAP * 0.0.0.0, EXCLUDE 127.0.0.1"
--dns-prefetch-disable
--disable-quic
--force-webrtc-ip-handling-policy=disable_non_proxied_udp
```

代理是项目硬约束，不属于自动优化项。控制器可以提示代理端口未监听，但不会自动关闭代理、改成直连、轮换节点或改变这些参数。

### 本地中转架构

```text
Android 手机浏览器
    ↓  局域网 HTTP / WebSocket，带访问令牌
Windows 控制器，默认 TCP 8765
    ↓  仅本机连接
Edge 动态 DevTools WebSocket
```

手机不直接连接 Edge 调试端口。不要把动态调试端口或 8765 映射到公网。

## 2. 核心能力

### v6.8.0 新增：静止画面高清化（默认开启）

阅读长对话时，连续 JPEG 的小字容易发糊。页面持续无新帧且无任何输入超过阈值（默认 1.5 秒）后，控制器自动补拍一张**无损 PNG** 替换最后一帧；一有点击、滚动、输入或新画面就立即回到实时流。

补拍只用 `Page.captureScreenshot` 和 `Page.getLayoutMetrics`，**不执行页面脚本、不启用 Runtime/DOM、不激活窗口、不注入任何内容**，对网页完全不可见，因此不增加任何页面可观测信号。可用 `idleSharpenEnabled` 关闭。

### v6.8.0 新增：电脑剪贴板桥（默认开启）

在输入面板点“电脑剪贴板”，可以**显式、单次、双向**地在手机与 Windows 剪贴板之间搬运文本：

- **从电脑读取**：取回电脑剪贴板文本（再复制到手机，明文 HTTP 下提供长按选中兜底）；
- **写入电脑**：把手机文本写入电脑剪贴板，电脑上直接粘贴。

读写都只在点击时发生一次，通过 PowerShell `Get-Clipboard`/`Set-Clipboard` 实现，完全在浏览器之外，不轮询、不自动同步、不落盘、不解析、不记录内容，超限明确拒绝。可用 `clipboardBridgeEnabled` 关闭。

### v6.8.0 新增：手机专用窗口（可选，默认关闭）

在设置里开启“手机专用窗口”后，控制器在**同一 Edge 配置**下新开一个独立窄窗口专供手机操作（同 Cookie、同登录、同扩展），**电脑主窗口完全不受影响**，退出即关窗，不再有“窗口恢复失败”。标签面板与标签跟随只作用于专用窗口。提供“关闭专用窗口及其中标签”按钮。

### ChatGPT / Claude 严格人工模式

默认在以下域名自动启用：

```text
chatgpt.com
chat.openai.com
auth.openai.com
claude.ai
claude.com
```

严格人工模式的目标是减少控制器自身产生的不必要页面干预，同时保留实际使用体验。它不是反检测方案，也不能保证网站不会要求验证。

启用后：

- 保留原 Windows Edge 的 User-Agent、平台、扩展和浏览器状态；
- 不修改 `navigator.webdriver`；
- 不伪造 Android User-Agent、Canvas、WebGL、字体或请求标头；
- 不使用无头浏览器或 `--enable-automation`；
- 使用真实 Edge 窗口的窄布局，而不是给目标网页套完整手机设备指纹；
- 普通点击映射为鼠标点击；
- 普通上下拖动映射为滚轮；
- 键盘输入只在用户明确输入时发送；
- 默认不长期启用 Runtime、DOM 或文件选择拦截；
- 不自动读取页面正文、回答内容或聊天列表；
- 不自动发送、重试、滚动读取、批量操作或处理验证码。

手机设置中可选择：

- **自动**：仅在上述 ChatGPT / Claude 相关域名启用；
- **始终启用**：整个控制会话都使用严格人工模式；
- **关闭**：恢复通用手机触摸和设备仿真模式。

### 原生触摸作为明确的临时兼容通道

严格人工模式默认使用桌面鼠标、滚轮和键盘。遇到地图、Canvas、滑块、拖拽排序或其他必须连续触摸的控件时，可以点：

```text
临时切换为原生触摸
```

全屏悬浮工具中也有相同切换。使用结束后可恢复桌面鼠标／滚轮。

### Windows UI Automation 标签跟随

v6.7.0 默认开启电脑 Edge 前台标签跟随，但实现方式不再是周期性附加网页并执行 `document.hasFocus()`：

- PowerShell 辅助进程使用 Windows UI Automation 读取前台窗口；
- 只在前台进程确实是 `msedge.exe` 时读取选中的标签标题和地址栏；
- 控制器用标题／地址与 CDP 标签列表进行保守、唯一匹配；
- 电脑切换到资源管理器、播放器或其他应用时，手机保持当前 Edge 标签；
- 匹配不唯一时保持当前标签，不猜测切换；
- 自动跟随不会反向抢占电脑焦点。

可在设置中关闭。关闭后 UIA 监视器停止，手机只在用户从“标签”面板主动选择时切换。

### CDP 能力按需启用

严格人工模式空闲时只保留网页显示和人工输入必需的能力：

```text
Target
Page
Input
screencast
```

以下能力改为按需使用：

- `Runtime`：只在用户点“按需读取当前环境”等明确功能时进行一次性调用；
- `DOM`：不在普通浏览时常驻；
- 文件选择拦截：只有用户先点控制器“上传”后临时启用。

### 校准标记只画在手机控制页面

三点快速校准、五点边缘校准、蓝色触点和红色命中点都绘制在手机控制器自己的覆盖层中，不向 ChatGPT 或 Claude 页面插入测试 DOM。

仍分别保存：

```text
普通模式 · 竖屏
普通模式 · 横屏
全屏 · 竖屏
全屏 · 横屏
```

### 电脑文件上传默认按修改时间排序

点击网页上传控件前，先点手机控制器顶部或全屏工具中的“上传”。控制器会在 90 秒内临时武装文件选择拦截；随后再点网页自己的上传按钮。

手机上传面板默认进入“电脑文件”，并按：

```text
更改时间：最新优先
```

排序。可以浏览桌面、下载、文档、图片、视频和磁盘，也可切换名称、大小或修改时间排序。选择的是电脑上的原始文件路径，不需要先传到手机。

## 3. 系统要求

- Windows 10 或 Windows 11；
- Microsoft Edge；
- Node.js `22.16.0` 或更高版本；
- Android 手机浏览器；
- 手机与电脑在可互相访问的同一局域网；
- 本地代理监听 `127.0.0.1:7897`。

## 4. 第一次安装与启动

建议解压到新目录，例如：

```text
G:\edge-phone-cdp-inherit-live-v6.8.6
```

不要覆盖唯一可用的旧版本。

### 第一步：安装／自检

双击：

```text
安装依赖.cmd
```

发布包已经包含 `ws` 依赖，通常不会联网下载。脚本会检查 Node.js 版本、JavaScript 语法和自动测试。

### 第二步：开放局域网端口

以管理员权限运行：

```text
允许防火墙.cmd
```

规则只开放控制器 TCP 8765，并限制为本地子网。不要为 Edge 动态调试端口创建入站规则。

### 第三步：启动

双击：

```text
启动.cmd
```

脚本会：

1. 读取原 Edge 配置；
2. 检查固定代理端口；
3. 关闭旧 Edge，使代理参数和动态调试状态由本次实例接管；
4. 删除失效的旧 `DevToolsActivePort`；
5. 使用原配置和固定代理启动 Edge；
6. 启动手机控制服务；
7. 自动等待并读取新的动态调试 WebSocket。

Edge 中应保持：

```text
edge://inspect/#remote-debugging
```

页面里的：

```text
Allow remote debugging for this browser instance
```

处于勾选状态。

控制台会显示手机地址，例如：

```text
http://192.168.0.21:8765/#token=……
```

手机打开该完整地址。不要继续使用旧版本标签页或旧令牌。

## 5. 日常操作

### 点击和滚动

严格人工模式：

- 轻点：鼠标点击；
- 上下拖动：滚轮滚动；
- 复杂拖拽：临时切换原生触摸。

通用模式：

- 默认保留直接触摸与拖拽；
- 可在设置中改为智能轻点与滚动。

### 输入

先点远程网页输入框，再点控制器“输入”，使用 Android 输入法。回车、退格、Tab 和 Esc 有独立按钮。

### 标签页

- 顶部“标签”可查看、选择和关闭 Edge 网页标签；
- 手机主动选择标签时，可以把该标签带到电脑前台；
- 默认 UIA 跟随电脑在 Edge 内的真实标签切换；
- 电脑切到其他 Windows 应用时，手机不会误切；
- 匹配不唯一时可手动选择。

### 浏览历史

“标签”面板中同时提供：

- 当前标签页的精确后退／前进会话历史；
- 原 Edge 配置的全局历史数据库，按日期和时间排序；
- 标题／网址搜索；
- 在当前标签或新标签打开。

### 上传电脑文件

正确顺序：

1. 点控制器“上传”；
2. 在 90 秒内点网页自己的“添加附件／选择文件”；
3. 手机面板选择“电脑文件”；
4. 进入目录，选择文件；
5. 点“使用已选电脑文件”。

这是有意的两步操作：普通浏览时不长期拦截文件选择器，只有用户明确准备上传时才临时启用。

### 全屏

进入沉浸全屏后，右侧保留可拖动的 `⋮` 工具，包含：

```text
后退  前进  刷新
标签  输入  上传
鼠标/触摸  校准  设置  退出全屏
```

工具是手机本地悬浮层，不占用远程网页布局。

### 校准

优先保持边缘比例为 `100%`，只调整水平和垂直位移。只有中央准确、越靠边越偏时才调整比例。

建议顺序：

1. 重置当前配置；
2. 打开点击测试；
3. 用方向微调让红色命中点与蓝色实际触点重合；
4. 普通竖屏调好后，进入全屏竖屏复制并单独微调；
5. 三点快速校准只拟合位移；
6. 五点边缘校准属于实验功能。

## 6. 设置建议

### ChatGPT / Claude

```text
严格人工模式：自动
输入：桌面鼠标／滚轮
标签跟随：开启（Windows UI Automation）
网络画面策略：自动或实时优先
```

地图、Canvas、滑块和拖拽时临时切换原生触摸。

### 只想完全手动切标签

关闭：

```text
跟随电脑当前标签页
```

这会停止 UIA 辅助进程，不会回退到网页 Runtime 轮询。

## 7. 连接与恢复

- Edge 意外退出时，控制器可以使用原配置和固定代理自动重启；
- 每次连接前都会验证 `DevToolsActivePort`，不会盲信残留文件；
- 同一时间只保留一个 CDP 重连流程；
- 手机 WebSocket 使用连接代际，旧连接迟到事件会被忽略；
- 静止网页没有新帧不会被误判为黑屏；
- 画面故障按“保留最后帧 → 截图 → 重启 screencast → 重附加标签 → 重连浏览器”分级恢复；
- 手机处理不过来时只保留最新画面，不积压旧帧。

要彻底关闭 Edge，先运行：

```text
停止控制器.cmd
```

否则在 `autoRestartEdge=true` 时，控制器会把意外关闭的 Edge 重新拉起。

## 8. 诊断

运行：

```text
检查环境.cmd
```

可检查 Node.js、代理、8765、Edge、动态端口和防火墙。

运行：

```text
导出诊断包.cmd
```

会收集控制器日志、端口状态、Edge 进程、动态端口文件、网络和防火墙摘要。诊断包不应包含账号 Cookie、网页正文或访问令牌明文。

## 9. 安全与使用边界

v6.8.0 明确不实现：

- 自动提交提示词；
- 自动重发上一条消息；
- 自动抓取、OCR 或保存回答正文；
- 批量对话或多账号并行；
- 验证码或机器人验证处理；
- 频率、额度或安全限制绕过；
- 随机鼠标轨迹、随机打字延迟等“伪装真人”；
- User-Agent、webdriver、Canvas、WebGL 或请求标头伪装。

控制器仍通过 CDP 附加 Edge。严格人工模式只能减少不必要的页面脚本、DOM、设备仿真和后台轮询，不能承诺任何网站永远不要求验证。v6.8.0 的静止补拍、剪贴板桥和专用窗口都遵守同一边界：不读取聊天正文、不解析页面、不注入 DOM、不激活窗口、不轮询。所有操作应由用户本人实时发起，并遵守所使用网站的条款和限制。
