# v6.8.4 更新说明

发布日期：2026-07-20

v6.8.4 是针对 v6.8.3 新增"页面缩放"功能的一次小型跟进修复，源于真机反馈。所有硬约束（固定代理及五个启动参数、继承原 Edge 配置、严格人工模式、禁止自动化/抓取/伪装、"零页面信号"边界）保持不变。

## 页面缩放扩展到桌面显示模式

v6.8.3 的"手机页面缩放"只对**手机**显示模式生效；手机处于**桌面**显示模式时调整缩放会被拦截并提示"缩放设置只影响手机显示模式的网页"，设置不生效——真机反馈正是这种场景。本版把设置项改名为"**页面缩放**"，同一套倍率（90%–150%，默认 100%）对两种显示模式统一生效：

- **手机显示模式**：行为与 v6.8.3 相同——仿真视口按倍率缩小，同屏显示即等比放大内容；
- **桌面显示模式**（本版新增）："桌面网页宽度"按倍率折算后再仿真。例如宽度 1280px、缩放 125% 时实际以 1024px 渲染，同屏内容放大约 25%；折算结果夹在服务端接受的 [480, 2560] 区间内，与"桌面网页宽度"滑块自由组合；
- **严格人工模式**：仍不受影响——保持真实桌面 Edge 窗口，其缩放由 Edge 自身按站点记忆（调整时提示不变）。

桌面模式下的拦截提示已移除，调整后直接生效并提示"页面缩放已设为 X%"。存储键与元素 id 不变，v6.8.3 已保存的缩放偏好升级后继续有效。

## 验证

- 真实 Chromium 端到端验证三段全部通过：手机模式 125%（视口 500 → 仿真 400）、桌面模式 125%（1280 → 1024，mobile:false）、调回 100%（恢复 1280）；
- `npm run check`、全部 16 个单元/模拟套件、浏览器 UI 测试、真实 Chromium CDP 集成测试、发布 ZIP 解包自检通过；
- CI 的 Ubuntu 与 Windows（真实 Microsoft Edge CDP、真实 PowerShell、真实剪贴板）作业全绿后合并发布。

其余行为与 v6.8.3 完全一致；v6.8.3 的完整修复清单见 `更新说明-v6.8.3.md`，真机验收边界见 `发布验证报告.md`。
