# v6.8.1 更新说明

发布日期：2026-07-19

v6.8.1 是 v6.8.0 的缺陷修复版本，不新增功能。本次对整个仓库做了一轮深度 bug 排查（服务端、手机前端、共享库、启动脚本与 CI/打包流水线分别独立审查后逐条核实），共确认并修复 23 处缺陷。所有硬约束（固定代理 `http://127.0.0.1:7897` 及五个启动参数、继承原 Edge 配置、严格人工模式、禁止自动化/抓取/伪装、v6.8 三项能力的"零页面信号"边界）保持不变。

## 1. 高危修复

- **CDP 恢复流程可重入死锁（server.js）**：`recoverFrames` 的函数体在 `lifecycleQueue` 内运行，而其第一步 `ensureConnected` 又会把 `connect()` 排进同一队列、排在自己身后，形成永久互等。Edge 在命令入队窗口期崩溃即可触发，之后所有重连、看门狗恢复和手机命令全部悬挂，只能重启控制器。现改为先建立连接、再把恢复动作入队。
- **控制器重启后手机画面永久冻结（public/app.js）**：前端的帧 epoch/序号排序门槛只增不减且跨重连保留，而服务端重启后计数从头开始，重启后的每一帧都被判为"过旧"而静默丢弃，只能刷新页面恢复。现于 WebSocket 连接成功时重置排序门槛（断线期间的旧画面仍保留显示）。
- **正式发布 ZIP 安装自检必失败（scripts/package-release.mjs）**：打包时排除了 `node_modules/.package-lock.json`，而安装器自检运行的 `project-integrity.test.js` 必读该文件，导致所有使用官方发布包的用户看到"安装失败"。现随包携带该文件，并在发布工作流中新增"解包 ZIP → 在包内跑完整测试"的发布门禁。

## 2. 中危修复

- 关闭后台标签前不再预先附加到该标签（原行为会把手机画面先切到即将关闭的标签、随后落到任意标签）；
- 控制权晋升定时器只在控制者本人回归时取消（原先任何设备连入都会取消晋升，控制权悬挂在已离线设备上，所有人只读）；
- 手机端令牌被轮换后不再无限 401 重连：连接从未成功打开时先校验令牌，正常弹出令牌输入框；
- 视口同步增加 4 秒看门狗：服务端恰逢 Edge 重启而丢弃 `viewport` 命令时，不再永久丢帧、拒绝触摸；
- 启动/停止脚本不再用 `/health` 中不存在的 `pid` 字段校验控制器（该回退分支永远失败，`npm start` 方式启动的控制器会被误判为"端口被其他程序占用"）；
- 7 个 `.cmd` 启动器去除 UTF-8 BOM（cmd.exe 不识别 BOM，每次双击都先报一行 `'∩╗┐@echo' 不是内部或外部命令`），并加入回归断言；`.ps1` 仍按要求保留 BOM；
- 发布工作流手动触发时显式指定 `target_commitish`，标签不再打到默认分支 HEAD 而与构建产物脱节；
- UIA 标签跟随的地址归一化不再把无协议的 `host:port`（如 `localhost:3000`、NAS/路由器地址）误解析成自定义协议（原行为导致这类地址的标签跟随完全失效），并补充回归测试。

## 3. 低危修复

- `Page.startScreencast` 持续失败时按 15 秒节奏重试且只在首次失败提示（原为每秒重试并向手机每秒弹一条相同警告）；
- 上传中断时监听文件流 `close` 事件，背压等待不再永久悬挂并泄漏缓冲区；
- 专用窗口的 `windowId` 查询遇 CDP 瞬态超时不再被误判为"窗口已关闭"（原行为会重复创建专用窗口），先经 `Browser.getWindowBounds` 确认再清除记录；
- UIA 监视器 spawn 失败（只触发 `error` 不触发 `exit`）后恢复自动重启；
- 剪贴板"写入超时"不再误报为"读取超时"；
- 三处重画路径不再覆盖未确认的排队帧（原行为使服务端空等 700ms 应答超时并误判网络拥塞、临时降画质）；
- 手势被取消（切后台、导航切换等）时补发被推迟的视口同步；
- 启动脚本改用 `Stop-Process` 关闭残留 Edge（PowerShell 5.1 下 `taskkill` 的 stderr 重定向会中断整个启动脚本）；
- 打包脚本统一把路径归一化为 `/`（原先在 Windows 上因反斜杠使全部排除规则静默失效，两平台产物不一致）；
- 防火墙脚本按通配名清理本工具创建过的所有旧规则（原先改过 `controllerPort` 后旧端口一直对局域网开放）；
- CI 的脚本解析门禁改用真实 Windows PowerShell 5.1（原用 pwsh 7 解析，会放过 5.1 不支持的语法）。

## 4. 验证

- `npm run check`、全部 16 个单元/模拟测试、前端浏览器 UI 测试、真实 Chromium CDP 集成测试全部通过；
- 本地执行发布打包并在解包后的 ZIP 内完整跑通自检（对高危第 3 项的端到端回归）；
- CI 的 Ubuntu 与 Windows 作业（真实 Microsoft Edge CDP、真实 PowerShell 解析、真实剪贴板往返）全部通过。

详细验证边界见 `发布验证报告.md`；Windows/Android 真机验收项与 v6.8.0 相同。
