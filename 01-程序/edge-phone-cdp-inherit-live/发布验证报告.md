## 1. 验证结论

v6.8.5 为真机反馈版本：画面清晰度端到端像素对齐（自动档引入清晰级、2.5× 采集、快降慢升）与输入体验（实时同步远程输入框、输入面板输入法避让、退格长按连发），改动经 8 视角代码审查后发布，详见 `更新说明-v6.8.5.md`。本版另新增实时同步端到端 17 项与按键连发 4 项真实 Chromium 验证（见更新说明第三节）。其 JavaScript 语法、单元测试、静态页面结构、项目完整性、模拟浏览器级 CDP、重连恢复、Windows UI Automation 匹配逻辑（含本版新增的不透明协议归一化与边界前缀匹配回归）、严格人工模式、静止画面高清化与剪贴板桥单元测试，以及真实 Chromium CDP 集成（含静止 PNG 补拍和手机专用窗口）均在本 Linux 容器通过。本次另在本容器实际执行发布打包，并在解包后的 ZIP 内完整跑通 `npm run check` 与全部单元/模拟测试（对"发布包安装自检"的端到端验证）；该解包自测已固化为发布工作流的门禁步骤。

与 v6.7.0 构建容器不同，本容器的 Chromium 未加载 `URLBlocklist` 管理策略，因此 `test/browser-ui.test.js` **实际运行并通过**（不再是跳过）。

Windows 专属 UI Automation 辅助脚本、真实 Microsoft Edge 原配置继承、固定本地代理、Android 手机行为，以及手机专用窗口“新标签落在专用窗口”的落点行为无法在 Linux 容器中最终验证，已明确列入真机验收。正式发布 ZIP 的打包、内部 SHA-256 清单和外部 ZIP SHA-256 由 GitHub Actions 在打 tag 时生成（见第 3 节）。

**GitHub Actions 已覆盖的真机项**：Windows 作业现用**真实 Microsoft Edge**（Edge 即 Chromium 内核）跑真实 CDP 集成、用真实 `Get/Set-Clipboard` 跑剪贴板往返、用真实 PowerShell 解析全部脚本并跑完整测试套件；Ubuntu 作业跑真实 Chromium 集成与前端 UI 测试。这些此前需人工在 Windows 上验收的项，现已变成每次提交的自动回归。仍需人工的只剩：真实 Edge 原配置/登录状态继承、UI Automation 控件树、固定代理实际联网、专用窗口“新标签落点”（无头模式不复现窗口激活）和 Android 真机体验。

自 v6.8.0 起的安全收尾（SEC-003 / SEC-004）继续生效：WebSocket 令牌由 Sec-WebSocket-Protocol 子协议携带、不进入 URL/日志；`/health` 收敛为存活性最小字段（v6.8.1 起启动/停止脚本也不再依赖其中的进程号）；自动生成的令牌支持一键在线轮换。

## 3. 发布打包（CI 驱动）

v6.8.0 起，正式发布 ZIP 不再手工打包，而是由 `.github/workflows/` 中的发布流程在打 tag（本版为 `v6.8.5`）时自动完成：

```text
从 02-AI交接/ 复制交接材料到 ZIP 内 AI交接/
生成内部 文件校验清单-SHA256.txt
打包 edge-phone-cdp-inherit-live-v6.8.5.zip
解包 ZIP 并在包内跑完整测试（v6.8.1 起的发布门禁）
计算 ZIP 的 SHA-256
上传到 GitHub Release
```

打包脚本 `scripts/package-release.mjs` 也可在本地运行以复现产物。CI 排除 `.git`、`logs/*.log`、`data/access-token.txt`、`data/history-snapshots/`、诊断包、上传临时文件和 `test/fixtures/ui-preview.png`。

## 5. 严格人工模式验证点

测试确认：

- 只对配置中的 ChatGPT/Claude 精确域名和子域启用严格模式，普通公司域名不会被泛化；
- 严格模式空闲附加不执行 `Runtime.enable`、`DOM.enable` 或长期 `Page.setInterceptFileChooserDialog`；
- 用户未点击环境审计前不执行 `Runtime.evaluate`；
- 默认轻点/滚动使用桌面鼠标语义；原生触摸只有用户显式开启后才启用；
- 严格模式不调用 User-Agent、Client Hints、自动化标志或指纹覆盖；
- 上传拦截由用户点控制器“上传”后短暂启用，完成、取消、换页或超时后关闭；
- 退出严格模式会尝试恢复真实 Edge 窗口；
- 校准标记和命中测试只存在于手机控制页面，不向目标网页注入校准 DOM。

## 6. v6.8.0 新增能力验证点

### 静止画面高清化（UX-008）

`idle-sharpen.test.js` 与真实 Chromium 集成确认：

- 页面无新帧且无输入超过阈值后，用 `Page.captureScreenshot(format=png)` 补拍并以 `source=idle-sharpen`、`contentType=image/png` 发布；
- 补拍帧本身不再触发补拍（不循环）；
- 手指按住（`touchActive`）期间不补拍；抬手并再次静止后可再次补拍；
- 补拍过程不启用 `Runtime`/`DOM`，不激活窗口。

### 电脑剪贴板桥（CLIP-001 / CLIP-002）

`windows-clipboard.test.js` 确认：

- 读取取回完整 UTF-8 文本；写入经 stdin 原样传给外部命令，不落盘；
- 超过字符上限时明确拒绝，且不触碰剪贴板（不截断）；
- WS `clipboardGet`/`clipboardSet` 为请求-应答式，无轮询路径；`hello` 正确声明 `clipboardBridge` 能力；
- 非 Windows 且未配置替代命令时明确报错。

### 手机专用窗口（WIN-001）

`windows-edge-uia.test.js` 与真实 Chromium 集成确认：

- `chooseTargetFromUia` 的 `allowedTargetIds` 之外的前台标签不被跟随（即使地址精确匹配），集合内正常唯一匹配；
- 专用窗口与主窗口 `windowId` 不同；标签面板只列出专用窗口内的标签，不含主窗口标签；
- 关闭专用窗口后其中所有标签被清理、原主窗口标签存活。

## 8. Windows / Android 真机验收建议

1. 代理关闭时确认 Edge 不会静默直连；
2. 启动后确认原 Edge 登录状态、扩展和历史仍在；
3. Windows 在 Edge 标签 A/B 间切换 50 次，手机应保守跟随；
4. Windows 在 Edge 与资源管理器/其他应用间切换 30 次，手机不得误切标签；
5. ChatGPT/Claude 普通模式与沉浸全屏分别测试点击、滚动、输入、临时原生触摸和校准；
6. 测试电脑文件上传、修改时间排序和 90 秒上传武装超时；
7. 开启手机专用窗口，确认电脑主窗口不受影响、新标签落在专用窗口、关闭按钮只清理专用窗口；
8. 在 ChatGPT/Claude 页面点复制后用剪贴板桥“从电脑读取”，再“写入电脑”回粘，确认双向一次性、不自动同步；
9. 连续使用 10 分钟检查画面不积压、静止后补拍清晰、断线后不补发旧输入；
10. 导出诊断包并确认不含访问令牌、Cookie、网页正文和上传文件内容。
