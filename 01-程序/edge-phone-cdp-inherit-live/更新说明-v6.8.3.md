# v6.8.3 更新说明

发布日期：2026-07-20

v6.8.3 围绕 Android 手机＋Windows Edge 的真机使用反馈修复，核心是彻底解决"ChatGPT 输入区按钮点不中"的坐标单位问题（真机已确认修复），并附带若干体验修正与一项小功能。所有硬约束（固定代理及五个启动参数、继承原 Edge 配置、严格人工模式、禁止自动化/抓取/伪装、"零页面信号"边界）保持不变。

## 一、点击可靠性（本版核心）

**CDP 坐标单位边界统一。** Chrome DevTools Protocol 三个输入接口的坐标单位不同：`Input.dispatchMouseEvent` 与 `Input.dispatchTouchEvent` 接收 **CSS 像素**，`Input.emulateTouchFromMouseEvent`（dev 仿真通道）接收 **DIP**。此前的换算基准 `contentDip ÷ pageScaleFactor` 只除得掉捏合缩放、除不掉浏览器缩放（Ctrl +/-、Edge 按站点记忆，此时 pageScaleFactor 仍为 1），DIP 值被当作 CSS 像素送入接口，形成乘性偏差：页面中部整体偏移，底部小按钮（ChatGPT 的 "+"、模型选择器、发送箭头）被推到视口之外而完全无法点击；dev 仿真因单位恰好正确而一直可用。修复后：

- 归一化坐标一律直接乘服务端**实时** `Page.getLayoutMetrics` 的 CSS 视口（6 秒新鲜窗口＋目标页匹配校验），帧上下文换算仅作回退；应用于轻点、手势与滚轮的落点/位移；
- 轻点、手势起点、滚轮入口前置刷新布局指标（内部 900ms 缓存与单飞，代价可忽略）；
- dev 仿真保持 DIP 直通；
- 新增**单位边界回归断言**：模拟浏览器缩放 125%（DIP 680×1056、真实 CSS 544×845），归一化中点必须注入 CSS 中点 (272, 422.5)±1——修复前的历代代码在此断言下全部失败。

配套修复与工具：

- **点击序列拟真化**：严格模式默认点击改为"悬停 → 按下 → 保持 60ms → 抬起"（此前为零时长、无悬停的按下-抬起，会被基于 pointer 时序的组件丢弃）；原生触摸轻点按压 16ms → 60ms；
- **点击测试远端元素探针（tapProbe）**：点击测试模式下每次点击，一次性、只读、不注入 DOM（与环境审计同一边界）地回报"换算后将要点击的位置"下的元素链（标签/aria/禁用态/pointer-events/矩形）与视口状态；坐标落在视口外时明确提示。本地十字只能验证手机侧映射，探针补上了服务端这一段的可观测性——正是定位本次根因的工具；
- **严格模式开放 dev 仿真通道**：开启"临时原生触摸"后，输入模式选择器解禁，可在原生触摸与 dev 仿真之间选择。

**升级提示：此前为补偿旧偏差手工调整过点击校准的用户，请在更新后清空校准（恢复默认）**——旧校准值在抵消已被移除的误差，沿用会反向过校准。

## 二、连接与模式切换

- **同机双控制页互踢修复**：clientId 为设备级，第二个控制页实例（新标签页/PWA/后台旧页）连接时会以 4001 挤掉旧连接，而旧页面此前会立即自动重连再挤回去，形成约 1 秒周期的"一直断开又连接"。现在被挤掉的页面进入停驻状态（显示原因与"重新连接"按钮），一切自动重连路径停止，用户显式点击才接管；
- 手机每次连接重放偏好时，`manualCompatibility` 值未变化不再强制整套重建显示环境（消除重连时的画面抖动与日志风暴）；
- **退出严格模式后页面尺寸异常修复**：严格模式期间的 `mobile:false` 只是显示状态，不再写入存储偏好与手机 localStorage（此前会把用户的"手机/桌面"选择永久覆盖，退出后以桌面布局渲染手机宽度页面）；退出瞬间手机强制重发一次真实视口。

## 三、日常体验

- **键盘发送不再拉回输入法**：点"发送"按钮后不再强制聚焦文本框（安卓输入法随之收起）；回车发送保持焦点便于连续输入；
- **浏览历史默认开新标签**：主点击区改为在新标签页打开（保留当前页面，如正在使用的 ChatGPT 对话），右侧"此页"按钮用于替换当前页面；
- **新增"手机页面缩放"设置**（90%–150%，默认 100%）：把仿真视口按比例缩小、同屏等比放大内容，与 Edge 的 Ctrl+ 缩放同效；只影响手机显示模式的普通网页，严格人工模式与桌面显示模式不受影响。
