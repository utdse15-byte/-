# v6.8.2 更新说明

发布日期：2026-07-20

v6.8.2 是继 v6.8.1 之后的又一个纯缺陷修复版本，不新增功能。本版汇集了两轮深度排查的成果：一轮针对 v6.8.1 修复本身的对抗性复查与服务端并发／前端状态机／低覆盖区域专项（22 处），一轮针对外部安全审计逐条核实后确认属实的缺陷（22 处），共修复 44 处缺陷。所有硬约束（固定代理 `http://127.0.0.1:7897` 及五个启动参数、继承原 Edge 配置、严格人工模式、禁止自动化/抓取/伪装、v6.8 三项能力的"零页面信号"边界）保持不变。

## 一、授权与认证边界

- **电脑文件根失败关闭**：显式配置 `computerFileRoots` 但目录全部不可用（盘符离线、被删）时，不再静默回退到整个用户目录，改为失败关闭并给出明确错误——刻意收窄的授权范围绝不能因配置失效而放宽。
- **访问令牌一致性**：并发启动时的令牌竞争改为"输者采用赢者的令牌"，不再互相覆盖（旧行为会造成两进程与文件三方各持不同令牌）；令牌无法读取或持久化时中止启动，不再带着"重启即失效"的内存令牌继续运行；令牌轮换改为临时文件＋原子改名，崩溃或磁盘满不再留下空或半截令牌文件。
- **上传控制权**：二进制上传分块强制校验控制者身份，接管控制时立即终止并清理旧控制者的在途上传——此前控制权被接管后，旧控制者仍能继续把文件写入电脑。
- **浏览历史收紧为控制者专属**：与电脑文件、剪贴板同属"屏幕画面之外的本机数据"，只读端不再能取回本机浏览历史；前端只读端同步禁用历史入口。
- **静态文件真实路径复查**：在词法包含检查之外增加 `realpath` 复查，封堵 `public` 目录内符号链接指向外部文件的读取路径。

## 二、配置健壮性

- 配置文件语法错误、权限失败不再被静默当成"无配置"：仅"文件不存在"时使用默认值，其余情况明确报错并中止启动。
- 布尔配置只接受明确写法（true/false/1/0/yes/no/on/off）：手滑写错的 `"flase"`、`"disabled"` 等不再被当成"开启"；PowerShell 启动器同样改用严格布尔转换（`[bool]"false"` 在 PowerShell 里是真）。

## 三、连接、画面与视口

- **重连不再永久冻屏**：重连时重置帧 epoch／序号门槛，并给每帧打上"连接代号"——保留显示的旧画面帧不再把门槛抬回重启前的水平，新服务端的小编号帧不再被全部丢弃。
- **视口同步看门狗**：服务端在 Edge 重启窗口丢弃 `viewport` 命令后，画面与触摸不再永久锁死；看门狗只在控制者角色重发，只读手机不再陷入"重发被拒→修订号越追越远→每 4 秒弹错误"的死循环。
- **CDP 恢复不再可重入死锁**（v6.8.1 已修复的延续保障）：帧恢复先建立连接、再入队恢复动作。
- Canvas 上下文丢失后的恢复帧不再被去重门槛丢弃、画面不再保持空白。
- 帧发送失败后主动重泵排队帧，不再依赖后续事件偶然唤醒。
- 帧协议编码端拒绝空图像缓冲，与解码端共享同一不变式。
- 旧版 DIP 坐标与归一化路径统一做半像素内缩，恰在宽/高边界的点击不再落在最后一个像素之外。

## 四、并发与生命周期

- **看门狗重挂载不再泄漏 CDP 会话**：重挂载前显式分离旧会话并作废文件选择拦截，不再每次泄漏一个附加会话（可能挂着上传拦截、压住桌面的原生文件对话框）。
- `ensureConnected` 对不同目标的在途连接不再直接复用其结果，用户显式选标签的请求不再被静默丢弃。
- 上传结束等待补齐 `close` 监听，断线清理后消息队列不再永久悬挂泄漏。
- Edge 历史同步查询的 busy timeout 降到 300ms，持锁时尽快走快照路径，减少事件循环阻塞（帧泵／触摸冻结）。
- 日志流补 `error` 监听（磁盘满／杀毒锁定时不再打到未捕获异常）；关停时补清多个残留定时器，避免关停窗口内重新拨号 Edge。

## 五、标签跟随身份匹配

- 地址归一化保留不透明协议前缀（about:／edge:／data: 等），`about:blank` 不再与主机名恰为 `blank` 的网页相撞。
- URL 前缀匹配要求终止在路径段／查询边界，`/foo` 不再匹配 `/foobar`；地址栏逐字输入的半截地址（如单字符或输入中途）不再触发切换。
- UIA 监视器成为孤儿的问题修复：控制器被强制结束（停止脚本、关窗口）后，隐藏的 PowerShell 监视进程通过父进程存活轮询自行退出，不再常驻后台。

## 六、数据、资源与诊断

- WebSocket 单条消息上限提升到 8 MiB 以容纳最坏合法剪贴板输入（100 万个 CJK／表情字符经 UTF-8＋JSON 编码可超过 1 MiB，此前会在校验前直接断连）；手机端按字节预检并给出明确提示。
- 电脑文件多根重叠时按"最长匹配"归属（此前最宽的根会接管标签与上级导航）；文件选择先规范化去重再检查数量上限。
- 剪贴板能力按读／写方向分别暴露；剪贴板写入超时不再误报为"读取超时"。
- 电脑文件选择器在 Windows 上过滤 `$Recycle.Bin` 等知名系统条目（`showHidden` 时仍显示）。
- Edge 历史：空 `userDataDir` 不再误解析为进程工作目录；`databaseUpdatedAt` 取主库与 `-wal` 的较新修改时间。
- UIA 监视器 stdout 缓冲加上限，异常的超长无换行记录不再无限累积内存。
- 诊断包脱敏加强：配置导出按键名递归遮蔽（token／password／secret／credential／key）并遮蔽自定义剪贴板命令，补齐 `attemptedEndpoint`、环境审计 URL、UIA 前台标签标题，Edge 进程命令行遮蔽 URL 凭据。

## 七、脚本与 CI

- `检查环境.ps1` 改读 `manualCompatibility.desktopTabFollow` 的正确路径，UIA 状态面板不再恒为空；`followDesktopTabs` 默认值与服务端对齐。
- 10 个测试的服务器子进程改为只监听 `127.0.0.1`：Windows 安装自检不再弹防火墙对话框（误点取消会创建覆盖端口放行规则的阻止规则）。
- CI 的 PowerShell 脚本解析门禁改用真实 Windows PowerShell 5.1，不再放过 5.1 不支持的语法。

## 八、验证

- `npm run check`、全部 16 个单元／模拟测试（含本版新增的多项回归：失败关闭的文件根、最长根匹配、去重后限额、不透明协议归一化、边界前缀匹配、空帧不变式、DIP 边界内缩等）、浏览器 UI 测试、真实 Chromium CDP 集成测试、发布 ZIP 解包自检全部通过。
- CI 的 Ubuntu 与 Windows（真实 Microsoft Edge CDP、真实 PowerShell 解析、真实剪贴板往返）作业全绿。

真机验收边界与 v6.8.0／v6.8.1 相同，详见 `发布验证报告.md`。

## 附：本版未修复（已知限制）

以下外部审计项经核实后判断为与本工具"单实例、局域网、保守跟随"的设计前提冲突、已由既有机制缓解、或需架构级改造，本版未修改并记录在案：Edge 历史 WAL 快照的事务一致性（Chromium 独占锁使 SQLite 在线备份 API 无法打开）、超大目录同步枚举的事件循环阻塞（需 worker 线程化）、Windows 隐藏属性的逐项检查、OFFSET 分页的瞬时漏项、超 32 层深度上传路径、多进程共享临时目录的互删、按 IP 的认证限流。
