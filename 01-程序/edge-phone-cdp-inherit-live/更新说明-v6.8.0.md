# v6.8.0 更新说明

发布日期：2026-07-19

v6.8.0 在 v6.7.0 严格人工模式、UIA 标签跟随、按需上传和全局历史的基础上，新增三项**只优化体验、不增加任何页面可观测信号**的能力，并把仓库从“ZIP + 人肉同步交接文档”升级为“Git 单一事实源 + CI 自动打包”。所有固定代理、原 Edge 配置继承、禁止自动化/抓取/伪装的硬约束保持不变。

## 1. 静止画面高清化（UX-008，默认开启）

阅读长对话时，连续 JPEG 画面的小字容易发糊。v6.8.0 在页面**持续无新帧且无任何输入**超过阈值（默认 1.5 秒）后，自动补拍一张**无损 PNG** 截图替换最后一帧；任何点击、滚动、输入或新的 screencast 帧都会立即回到实时 JPEG 流。

设计边界（为什么它不构成可疑信号）：

- 只使用 `Page.captureScreenshot`（合成器层截图）与 `Page.getLayoutMetrics`（被动读取），**不执行页面 JS、不启用 Runtime/DOM、不注入任何内容**；
- 截图对网页完全不可见：不触发 `requestAnimationFrame`、不产生 `visibilitychange`、页面拿不到任何时序信号；
- **绝不为补拍激活或置顶窗口**（`noActivate`），避免产生 `focus`/`visibilitychange` 事件；
- 补拍失败被视为纯锦上添花，不进入重启/重连恢复流程；补拍帧本身不会再触发补拍。

可用 `idleSharpenEnabled` 关闭、`idleSharpenDelayMs` 调整触发延迟。

## 2. 电脑剪贴板桥（CLIP-001 / CLIP-002，默认开启）

在手机上让 ChatGPT/Claude 写的内容此前无法带入手机其他 App。v6.8.0 增加一个**显式、单次、双向**的剪贴板通道：

- **从电脑读取**：把 Windows 剪贴板文本取回手机（可再复制到手机剪贴板，明文 HTTP 下提供选中+长按兜底）；
- **写入电脑**：把手机文本框内容写入 Windows 剪贴板，电脑上直接粘贴。

设计边界：

- 读/写都**只在用户当次点击时发生一次**，没有任何轮询、监听或自动同步；
- 通过 PowerShell `Get-Clipboard` / `Set-Clipboard` 实现，**完全在浏览器之外**，不触碰目标网页、不使用 CDP Runtime；
- 内容不落盘、不写日志（只记录字符数）、不做任何解析或抓取；
- 超过大小限制（默认 100 万字符）时**明确拒绝，绝不静默截断**。

可用 `clipboardBridgeEnabled` 关闭。它与 HUMAN-003 禁止的“自动读取/抓取/批量导出”性质不同——主语始终是用户的当次动作，等价于人自己抄一遍。

## 3. 手机专用窗口（WIN-001，可选，默认关闭）

严格模式会把用户正在用的真实 Edge 窗口调成窄窗口，退出后只能“尽力恢复”。v6.8.0 增加可选的专用窗口模式：

- 控制器在**同一 Edge 配置**下新开一个独立窄窗口（同 Cookie、同登录、同扩展、同一 session）专供手机操作；
- **电脑主窗口完全不受影响**，退出即关窗，从根上消除“窗口恢复失败”问题；
- 标签面板与 UIA 标签跟随**只作用于专用窗口内的标签**；主窗口的标签活动不改变手机画面；
- 歧义时仍保守保持当前标签，不猜测切换。

它对平台而言就是同一账号的又一个普通窗口，不引入任何新指纹或“多设备并发”信号。可在设置中开启，并提供“关闭专用窗口及其中标签”按钮。

> 说明：真实 Windows Edge 会把新标签开在当前活动窗口，因此控制器创建前先激活专用窗口的锚点标签。此“新标签落点”行为需在 Windows Edge 真机验证；无头 Chromium 不复现窗口激活语义（已在验证报告中如实标注）。

## 4. 工程：Git 单一事实源 + CI 自动打包

- 目录去掉版本号后缀（`edge-phone-cdp-inherit-live-v6.7.0` → `edge-phone-cdp-inherit-live`），删除应用内与 `03-发布资料` 中重复的交接/发布文档副本，仓库成为唯一事实源；发布 ZIP 的 `AI交接/` 由打包脚本从 `02-AI交接/` 自动复制生成。
- `project-integrity` 改为从 `package.json` 读取版本，消除多处硬编码版本漂移。
- 新增 `.gitattributes` 统一 LF 存储、`.ps1`/`.cmd` 强制 CRLF 检出（配合 UTF-8 BOM）。
- 新增 GitHub Actions：在 Ubuntu 与 `windows-latest`（真实 PowerShell 解析 UIA helper、Edge 冒烟）上跑 check/test，Ubuntu 上跑真实 Chromium 集成；打 tag 时自动生成 ZIP + SHA-256 + 内部清单并发布到 Release。

## 5. 固定代理与原配置不变

```text
--proxy-server="http://127.0.0.1:7897"
--host-resolver-rules="MAP * 0.0.0.0, EXCLUDE 127.0.0.1"
--dns-prefetch-disable
--disable-quic
--force-webrtc-ip-handling-policy=disable_non_proxied_udp
```

不创建隔离 `user-data-dir`，继续继承原 Edge Cookie、登录状态、扩展、历史和权限。

## 6. 保留功能

v6.7.0 的严格人工模式、原生触摸临时通道、UIA 标签跟随、按需两步上传、四套校准、同帧归一化坐标、frameAck 最新帧优先、分级画面恢复、全屏悬浮工具、Edge 全局历史、电脑文件直接上传、单一控制权与 Edge 恢复全部保留。

## 7. 不包含的功能

不加入指纹伪装、stealth、webdriver 隐藏、随机人类行为、自动提示词、自动发送/重试、回答抓取、验证码或限制绕过、多账号和无人值守操作。剪贴板桥与静止补拍都不改变这一边界：它们不读取聊天正文、不解析页面、不注入 DOM。
