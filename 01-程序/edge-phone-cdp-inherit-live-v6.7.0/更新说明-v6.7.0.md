# v6.7.0 更新说明

发布日期：2026-07-19

v6.7.0 在 v6.6.0 的画面、坐标、全屏工具、标签、全局历史、电脑文件上传和断线恢复基础上，新增面向 ChatGPT 与 Claude 的**严格人工模式**，并把 AI 需求交接材料直接纳入发布包。

## 1. 严格人工模式

默认精确匹配：

```text
chatgpt.com
chat.openai.com
auth.openai.com
claude.ai
claude.com
```

不泛化整个 `openai.com` 或 `anthropic.com`。

严格模式会：

- 保持原 Windows Edge User-Agent、Client Hints、平台、语言、扩展和指纹环境；
- 清除控制器设置的移动设备指标和长期触摸仿真；
- 把真实 Edge 窗口调整到默认 560×960，以窄桌面窗口获得可用布局；
- 默认使用鼠标点击、滚轮和键盘；
- 提供用户明确开启的临时原生触摸；
- 空闲时不长期启用 Runtime、DOM 和文件选择拦截；
- 环境审计只在用户点击后执行一次；
- 校准标记只存在于手机控制页，不注入网站 DOM。

## 2. 按需文件上传

严格站点采用两步流程：

1. 点控制器顶部/全屏工具的“上传”，武装文件选择器 90 秒；
2. 在网页中点击上传按钮。

完成、取消、切换标签或超时后，文件选择拦截立即关闭。电脑文件继续默认按更改时间最新优先，并直接使用 Windows 原路径。

## 3. Windows UI Automation 标签跟随

默认 `followDesktopTabs=true`、`desktopTabFollowStrategy=uia`：

- 只在 Windows 前台应用为 Edge 时读取选中标签和地址栏；
- 唯一地址匹配优先，唯一标题匹配次之；
- 重复标题/地址时保持当前标签，不猜测；
- 电脑切到其他应用时不切手机标签；
- UIA 不可用时手动切换；
- 严格站点不自动回退到 `document.hasFocus()` 轮询。

## 4. 固定代理和原配置不变

```text
--proxy-server="http://127.0.0.1:7897"
--host-resolver-rules="MAP * 0.0.0.0, EXCLUDE 127.0.0.1"
--dns-prefetch-disable
--disable-quic
--force-webrtc-ip-handling-policy=disable_non_proxied_udp
```

不创建隔离 `user-data-dir`，继续继承原 Edge Cookie、登录状态、扩展、历史和权限。

## 5. 保留功能

- 通用网站的直接触摸与拖拽、原生触摸事件；
- 普通/全屏、竖屏/横屏四套校准；
- 同帧坐标、视口修订、顶部第一行命中；
- frameAck、最新帧优先、截图兜底和分级恢复；
- 全屏悬浮工具；
- Edge 全局浏览历史；
- Windows 电脑文件直接上传与手机文件备用；
- 单一手机控制权、断线释放和 Edge 恢复。

## 6. AI 交接材料

发布包内新增 `AI交接` 目录：

- 需求与风险平衡规范 v1.1（Markdown + DOCX）；
- 机器可读 YAML 基线；
- v6.7.0 实现对照表；
- 给下一位 AI 的提示词；
- 已知限制与后续路线。

## 7. 不包含的功能

不加入指纹伪装、stealth、webdriver 隐藏、随机人类行为、自动提示词、自动发送/重试、回答抓取、验证码或限制绕过、多账号和无人值守操作。
