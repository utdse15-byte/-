## 1. 验证结论

v6.7.0 的 JavaScript 语法、单元测试、静态页面结构、项目完整性、模拟浏览器级 CDP、重连恢复、Windows UI Automation 匹配逻辑、严格人工模式和真实 Chromium CDP 集成均通过。

发布目录中的真实 Chromium 集成测试连续运行三轮并通过。随后生成发布候选 ZIP，从全新目录解压，内部 84 个文件的 SHA-256 全部匹配；解压目录再次通过语法检查、单元/模拟测试和真实 Chromium CDP 集成。手机页面自动化因构建容器中的 Chromium 管理策略阻止全部 URL 而按设计跳过，不能写成通过。

Windows 专属 UI Automation 辅助脚本、真实 Microsoft Edge 原配置继承、固定本地代理和 Android 手机行为无法在 Linux 构建容器中最终验证，已明确列入真机验收。

## 2. 发布目录实际执行结果

执行：

```text
npm run check
npm test
npm run test:browser
npm run test:real   （连续三轮）
```

结果：

```text
npm run check        PASS
npm test             PASS
npm run test:browser SKIP（容器 Chromium URLBlocklist）
```

## 5. 严格人工模式验证点

测试确认：

- 只对配置中的 ChatGPT/Claude 精确域名和子域启用严格模式，普通公司域名不会被泛化；
- 严格模式空闲附加不执行 `Runtime.enable`、`DOM.enable` 或长期 `Page.setInterceptFileChooserDialog`；
- 用户未点击环境审计前不执行 `Runtime.evaluate`；
- 默认轻点/滚动使用桌面鼠标语义；
- 原生触摸只有用户显式开启后才启用；
- 严格模式不调用 User-Agent、Client Hints、自动化标志或指纹覆盖；
- 上传拦截由用户点控制器“上传”后短暂启用，完成、取消、换页或超时后关闭；
- 退出严格模式会尝试恢复真实 Edge 窗口；
- 校准标记和命中测试只存在于手机控制页面，不向目标网页注入校准 DOM。

## 6. Windows UI Automation 验证点

Node 单元测试确认：

- 地址和标题归一化；
- URL 精确/唯一前缀匹配；
- 标题精确/唯一前缀匹配；
- 多个相同 ChatGPT/Claude 标题时返回无匹配；
- 非 Edge 前台时保持当前标签；
- UIA 进程失败时状态可诊断并按间隔重启；
- 严格站点不自动回退到 `document.hasFocus()` Runtime 轮询。

PowerShell helper 已通过 UTF-8 BOM、CRLF、语法结构和项目完整性检查，但真实控件树必须在 Windows Edge 上验证。

## 10. Windows / Android 真机验收建议

1. 代理关闭时确认 Edge 不会静默直连；
2. 启动后确认原 Edge 登录状态、扩展和历史仍在；
3. Windows 在 Edge 标签 A/B 间切换 50 次，手机应保守跟随；
4. Windows 在 Edge 与资源管理器/其他应用间切换 30 次，手机不得误切标签；
5. ChatGPT/Claude 普通模式与沉浸全屏分别测试点击、滚动、输入、临时原生触摸和校准；
6. 测试电脑文件上传、修改时间排序和 90 秒上传武装超时；
7. 连续使用 10 分钟检查画面不积压、静止页面不闪烁、断线后不补发旧输入；
8. 导出诊断包并确认不含访问令牌、Cookie、网页正文和上传文件内容。
