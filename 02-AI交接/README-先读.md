# AI 交接目录：先读

本目录与 v6.8.0 程序一同发布。其他 AI 或开发者应按以下顺序阅读：

1. `Edge手机远程浏览器控制器-需求与风险平衡规范-v1.2.md`
2. `Edge手机远程浏览器控制器-需求基线-v1.2.yaml`
3. `v6.8.0实现对照表.md`
4. `已知限制与后续路线.md`
5. `历史问题与设计结论.md`
6. 程序根目录的 `README-中文.md`、`更新说明-v6.8.0.md`、`发布验证报告.md`

需要交给下一位 AI 时，可直接附上 `给下一位AI的提示词.txt`，但简短提示词不能替代完整规范。

事实源：仓库 `02-AI交接/` 是本交接资料的唯一副本；发布 ZIP 内的 `AI交接/` 由打包脚本从它复制生成，请勿另维护多份人肉同步副本。

## 最高优先级

- 固定代理及五个启动参数不可修改、绕过、轮换或建议关闭；
- 不使用隔离 Edge 配置，继承原 Cookie、登录、扩展、历史和权限；
- 项目只转发真人实时操作，不做自动发送、抓取、批量、验证码/限制绕过或指纹伪装；
- ChatGPT/Claude 使用严格人工模式和最小 CDP 面；
- 标签跟随默认 Windows UI Automation，不确定时保持当前标签；
- 静止画面高清化只做合成器层截图，不执行页面脚本、不启用 Runtime/DOM、不激活窗口；
- 电脑剪贴板桥只在用户当次显式触发，单次双向，浏览器之外实现，不轮询、不自动同步、不落盘、不解析正文；
- 手机专用窗口为可选（默认关闭），同配置新开独立窗口不动主窗口，标签面板与跟随只作用于专用窗口；
- 正式交付必须包含完整 ZIP、SHA-256、说明、验证报告和本交接目录。
