# Edge 手机远程浏览器控制器：需求基线、风险边界与实现规范

**面向 ChatGPT / Claude 真人操作场景**  
规范版本：1.1  
日期：2026-07-19  
当前实现基线：v6.7.0  
文档性质：后续开发、审查和 AI 交接的单一需求来源

> **给后续 AI/开发者的第一条指令：** 本文标为 **MUST** 的内容不得自行重解释。固定代理、原 Edge 配置继承、真人操作、禁止内容抓取/自动发送、禁止指纹伪装、UIA 标签跟随和完整 ZIP 交付均为硬约束。发现冲突时先说明冲突并征得用户确认，不得静默牺牲这些要求。

## 1. 项目目标与定位

本项目不是普通远程桌面，也不是 Selenium/Playwright 自动化、爬虫或所谓“反检测浏览器”。目标是在 Android 小屏幕上，以接近手机浏览器的方式查看并**实时人工操作** Windows 上的特殊 Microsoft Edge，同时保留电脑已有的代理、登录状态、扩展、历史和文件环境。

主要站点为 ChatGPT 与 Claude。v6.7.0 通过真实桌面 Edge 身份、真实窄窗口、鼠标/滚轮默认输入、最小空闲 CDP 能力、按需文件选择和 Windows UI Automation 标签同步，在可用性与最少页面介入之间取平衡。该设计不构成平台许可，也不能保证永不触发验证或限制。

## 2. 不可变约束

### 2.1 固定代理不可修改

以下五个参数是用户长期验证过的基础设施，不是排障变量：

```text
--proxy-server="http://127.0.0.1:7897"
--host-resolver-rules="MAP * 0.0.0.0, EXCLUDE 127.0.0.1"
--dns-prefetch-disable
--disable-quic
--force-webrtc-ip-handling-policy=disable_non_proxied_udp
```

后续 AI 不得建议关闭代理、改用直连、切换节点、轮换 IP、加直连回退或以“降低风险”为由删除任一参数。代理不可用时应失败关闭。

### 2.2 原 Edge 状态必须继承

- 不使用隔离 `--user-data-dir`；
- 使用原 Edge 用户数据目录和实际配置（例如 `Default` / `Profile 1`）；
- 保留 Cookie、登录状态、扩展、收藏夹、浏览历史、网站权限和 Windows 本地证书；
- 手机控制页不另外登录 ChatGPT/Claude，网站仍看到电脑上的同一 Edge 会话。

### 2.3 只能是真人实时操作

允许自动化的仅限基础设施：画面编码、旧帧丢弃、重连、窗口恢复、端口恢复、临时文件清理等。所有会改变网站状态的动作必须来自用户当下的手机操作。

## 3. 用户已确认的优先级

1. 固定代理和原 Edge 状态优先级最高，不接受用关闭代理或隔离配置解决问题。
2. 手机必须看到网页并能直接操作，不接受全桌面远控。
3. ChatGPT/Claude 以严格人工模式为默认；普通网页保留原生直接触摸体验。
4. 手动位移校准通常比自动拟合可靠；比例默认 100%，只在边缘误差随距离增长时调整。
5. 全屏必须可操作，保留悬浮工具和独立校准。
6. 上传默认选择电脑文件，默认按更改时间最新优先。
7. 历史应接近 Edge 侧栏的全局历史，而非仅当前标签的后退/前进。
8. Edge 内切标签可同步；Windows 切换其他应用不能误判为换标签。
9. 正式交付必须是完整压缩包，并与 AI 交接资料放在一起。

## 5. v6.7.0 风险—体验平衡方案

### 5.1 严格人工模式（ChatGPT / Claude 默认）

自动匹配的精确域名为：

```text
chatgpt.com
chat.openai.com
auth.openai.com
claude.ai
claude.com
```

严格模式行为：

- 清除控制器先前设置的手机设备指标和触摸仿真；
- 保留真实 Windows Edge 的 User-Agent、Client Hints、平台、语言、Canvas/WebGL、扩展和请求环境；
- 通过真实 Edge 窗口调整到默认 `560×960`，而非伪装 Android 指纹；
- 默认轻点为鼠标点击、上下拖动为滚轮、文字为明确的键盘/文本输入；
- 原生触摸由用户临时开启，用完再关闭；
- 空闲时不长期启用 Runtime、DOM 和文件选择拦截；
- 环境审计只能由用户点击后运行一次；
- 文件上传采用“先点控制器上传、再点网页上传按钮”的显式两步流程；
- 校准完全在手机控制页绘制，不注入目标网页；
- 默认由 Windows UI Automation 跟随电脑当前 Edge 标签。

### 5.2 通用原生触摸模式

非严格站点保留用户偏好的“直接触摸与拖拽 + 原生触摸事件”，可选手机布局、四套校准和全屏悬浮工具。严格站点的临时原生触摸开关不会改变通用模式偏好。

### 5.3 为什么不加入“反检测”功能

本项目不修改指纹、不伪造 webdriver、不随机鼠标轨迹或打字延迟，也不隐藏 DevTools。合理目标是减少不必要的页面介入并保留真实用户时序，而不是欺骗平台检测。

## 6. v6.7.0 技术架构

```text
Android 手机浏览器
        │  HTTP / WebSocket + 访问令牌
        ▼
Windows Node.js 控制器（TCP 8765，LocalSubnet）
        │  127.0.0.1 动态 DevTools WebSocket
        ▼
原 Microsoft Edge 配置 + 固定本地代理

Windows UI Automation 辅助进程
        │  只读前台窗口、选中标签和地址栏
        └──────────────► 控制器的标签匹配逻辑
```

### 6.1 严格模式的最小 CDP 面

空闲状态保持 `Target`、`Page`、`Input` 和 screencast/screenshot。`Runtime`、`DOM`、文件选择拦截只在明确功能需要时短暂启用。运行环境审计后不会长期保留 Runtime。

### 6.2 Windows UI Automation 标签同步

PowerShell 辅助进程读取前台窗口是否为 Edge、选中标签标题和可取得的地址栏值。控制器按“唯一地址精确/前缀 → 唯一标题精确/前缀”匹配 CDP 标签；多个标题相同的 ChatGPT/Claude 会话属于歧义，必须保持当前标签。

UIA 不可用、Edge 不在前台或 UI 树结构变化时，降级为手机手动标签选择，不在严格模式中自动恢复网页脚本焦点轮询。

### 6.3 画面与坐标

- 画面等比绘制；
- 手机显示完成后发送 `frameAck`，服务端只保留最新帧；
- 静止页面无新帧不等于故障；
- 卡住时按最后帧、截图、重启 screencast、重新附加、重连浏览器逐级恢复；
- 坐标绑定同一帧几何与视口修订；
- 手势期间冻结目标和几何；
- 普通/全屏、竖屏/横屏分别保存校准；
- 严格模式校准只影响控制器映射，不触碰网站 DOM。

### 6.4 文件、历史和隐私

- 电脑文件浏览器默认 `modified-desc`，只列出必要元数据；
- 用户确认后才把 Windows 原路径交给网页文件输入；
- 手机文件分块上传作为备用；
- Edge 全局历史从本地 History 数据库只读副本按需查询；
- 不持久保存网页帧、聊天正文或文件内容；
- 诊断包脱敏令牌和浏览器 WebSocket ID，但最近日志可能包含标题或网址，分享前需查看。

## 7. 平台条款与风险边界

- OpenAI 当前消费者条款限制自动/程序化提取数据或输出，并禁止绕过频率限制、限制和保护措施；本项目明确不实现这些行为。[R1]
- OpenAI 帮助文档列出异常位置/设备、使用模式突变和多个并发会话等警告原因，并在排障中提及代理和自动化工具。用户的代理是不可变硬约束，因此 v6.7.0 只通过稳定同一会话、单一控制者和最小 CDP 面减少其他变量，不承诺消除警告。[R2]
- Anthropic 消费者条款对通过自动或非人工方式访问 Claude 的限制更明确。真人通过 CDP 遥控是否被其认可没有公开明确结论；严格人工模式只能说明该项目不做无人值守、抓取、批量或保护绕过，不能消除条款不确定性。[R3]
- Microsoft 官方文档说明可通过 `edge://inspect` 连接正在运行的 Edge，并警告连接会继承当前登录账号、Cookie 等敏感状态；因此本项目只在本机连接 CDP，并要求可信控制器和局域网隔离。[R4]

本文不是法律意见。平台条款可能更新，重大发布前必须重新核对官方页面。

## 9. 已知限制

1. Windows UI Automation 辅助进程只能在真实 Windows Edge 上最终验证；不同 Edge 版本、语言、垂直标签页和无障碍树变化可能影响地址栏/标签识别。
2. 多个标签标题和地址完全相同时，保守策略会保持当前标签，需要手机手动选择。
3. 严格模式会调整真实 Edge 窗口大小，可能影响电脑端布局；退出时为尽力恢复，不保证在所有窗口管理状态下百分之百还原。
4. 严格站点的上传为显式两步操作；这是按需权限设计，不是网页自身行为。
5. CDP 画面是 JPEG/截图通道，不包含 Windows 原生文件窗口、扩展弹窗、浏览器菜单或系统音频。
6. 实际 Android 浏览器的全屏动画、厂商工具栏和网络延迟仍需用户设备验证。
7. 本版本无法承诺 ChatGPT/Claude 平台风险为零，也不代表平台对该访问方式给出许可。
