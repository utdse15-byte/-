# 已知限制与后续路线

版本：v6.7.0  
日期：2026-07-19

## 已知限制

### 1. Windows UI Automation 依赖真实 Edge 无障碍树

构建环境不是 Windows，无法实际执行 `helpers/edge-uia-monitor.ps1`。当前已完成 PowerShell 静态检查、Node 管理器单元测试和保守匹配测试，但以下因素需真机确认：

- Edge 版本和界面语言；
- 水平标签、垂直标签、标签分组；
- 地址栏无障碍名称和控件位置变化；
- 多窗口、相同标题和相同 URL 标签；
- 管理员/非管理员权限边界。

设计上的安全降级是：不确定就保持手机当前标签，不猜测，不启用严格站点 Runtime 轮询。

### 2. 严格人工模式不等于平台许可

v6.7.0 移除了不必要的设备混搭、常驻 DOM/Runtime 和页面脚本标签探测，也禁止无人值守、抓取和保护绕过。但 Claude 消费者条款对“自动或非人工方式”限制较明确，CDP 真人遥控是否被认可没有公开明确结论。不得把本版本宣传为“完全安全”或“平台允许”。

### 3. 真实窄窗口会改变电脑端 Edge

严格模式调整真实 Edge 窗口到默认 560×960。控制器会保存并尽力恢复原边界和最大化状态，但 Windows 多显示器、贴靠布局、虚拟桌面或异常退出可能导致恢复不完全。

### 4. 文件上传的两步流程是有意设计

严格站点先点控制器“上传”，再点网页上传按钮。它减少空闲文件选择拦截，但比普通浏览器多一步。不得为了省一步而恢复长期拦截。

### 5. CDP 只显示网页内容

无法显示/操作 Windows 原生文件对话框、Edge 顶部菜单、扩展弹窗、证书窗口、系统认证窗口，也不传输系统音频。电脑文件浏览器是对网页文件输入的替代路径。

### 6. 画面通道不是视频流

连续 JPEG 和截图适合聊天、表单和管理页；高速视频、游戏和大量 Canvas 动画可能不够流畅。不得通过积压每一帧换取“完整”，实时性优先。

### 7. 浏览器 UI 自动化测试在构建容器被策略跳过

容器 Chromium 由管理策略设置 URLBlocklist，`test/browser-ui.test.js` 按设计跳过。其他前端结构测试、模拟 CDP 和真实 Chromium CDP 集成均通过，但 Android 真机仍需验证。

## 建议的后续路线

### P0：Windows/Android 回归测试与诊断

- 增加 UIA 真实状态的诊断快照：前台进程、选中标签、地址值、匹配置信度和保持原因；
- 在 Windows 11 + Edge Stable 上建立 30 次应用切换和 50 次标签切换脚本化人工验收表；
- 在用户 Android 浏览器记录全屏、软键盘、旋转和网络延迟结果；
- 不记录聊天正文，只记录状态、延迟和错误代码。

### P1：UIA 兼容层

- 适配垂直标签和标签组；
- 针对中文/英文 Edge 无障碍名称建立多策略定位；
- 对多个窗口使用前台 HWND 和进程 ID 缩小候选；
- 保持保守匹配，不因体验压力放宽到可能误切。

### P1：严格模式可用性

- 在全屏工具中更明显显示“桌面鼠标”或“临时原生触摸”；
- 上传武装剩余时间和关闭原因可视化；
- 当窗口恢复失败时给出明确的一键恢复操作；
- 将严格模式域名配置界面化，但默认列表仍精确，不能泛化整个公司域名。

### P2：浏览历史与最近关闭

- 在本地会话数据有稳定、只读和低风险接口时增加“最近关闭”；
- 不逆向 Edge 云同步服务，不实现“其他设备标签”抓取。

### P2：发布工程

- 在 Windows CI 或可信虚拟机上执行 PowerShell 解析、UIA helper 启动和 Edge Stable 冒烟测试；
- 在 Android 浏览器测试环境可用时恢复手机页面自动化；
- 保持完整 ZIP、内部文件 SHA 清单、外部 ZIP SHA 和交接文档同步发布。

## 明确不进入路线图

- 反检测、stealth、指纹修改或 webdriver 隐藏；
- 随机鼠标/键盘行为；
- 自动提示词、自动重试、自动发送；
- 回答抓取、完整对话导出或批量爬取；
- 验证码、频率、额度或安全保护绕过；
- 多账号并发、IP 轮换或代理切换；
- 用关闭固定代理或隔离 Edge 配置换取兼容性。
